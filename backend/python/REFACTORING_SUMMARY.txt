## Key Features Implemented

### 📊 Data Storage

| File | Format | Refresh | Size/hour |
|------|--------|---------|-----------|
| bme688_data.csv | Timestamp, T, H, P, Gas | 2s | 1.8 MB |
| dds2_data.csv | Timestamp, SN, PPB, T, H, ADC | 1s | 2.2 MB |
| combined_data.csv | **Aggregated 60s windows** | 60s | 288 KB |

### 🔬 Calibration Logic

**VOC Calibration (DDS2):**
```
Warm-up: 15 minutes
├─ Collects 100+ samples
├─ Calculates baseline PPB average
└─ Calibrated PPB = |ppb_raw - baseline_ppb|
```

**AQI Calculation (BME688):**
```
baseline_resistance = initial gas resistance (clean air)
raw_aqi = (baseline_resistance / current_resistance) * 100
normalized_aqi = clip(raw_aqi * 5, 0, 500)
```

### 🔄 Data Aggregation (60s Window)

```python
combined_data.csv columns:
- timestamp
- temperature_avg (BME688)
- humidity_avg (BME688)
- pressure_avg (BME688)
- voc_avg (DDS2)
- voc_max (DDS2)
- voc_min (DDS2)
- voc_rate_change (PPB/minute)
- bme688_readings_count
- dds2_readings_count
```

### 🎯 JSON Output Format

```json
{
  "timestamp": "2026-03-25T14:30:00.123456Z",
  "temperature": 22.50,
  "humidity": 45.30,
  "pressure": 1013.25,
  "voc": 125.50,
  "aqi": 87.5,
  "data_quality": "excellent",
  "calibrated": true
}
```

### 🧵 Threading Architecture

```
Main Thread
├── BME688 Reader Thread (reads every 2s)
│   └── Appends to buffer → CSV
├── DDS2 Reader Thread (reads ~1s continuous)
│   └── Applies calibration → Appends to buffer → CSV
└── Aggregation Thread (60s interval)
    └── Computes averages → Saves combined_data.csv
```

### 📝 Error Handling

✅ **Graceful Degradation:**
- BME688 fails → VOC data available, no temperature/AQI
- DDS2 fails → Environmental data available, no VOC
- Both fail → System still runs, returns `data_quality: "poor"`

✅ **Automatic Recovery:**
- Serial reconnection on DDS2 failure
- I2C retry on BME688 failure
- Timeout handling for both sensors

## Usage Examples

### 1. Start System (Interactive Mode)

```bash
cd /home/hritik2144/foodmonitor/backend/python
python3 main.py
```

Then use commands:
- `r` - View current readings
- `s` - View system status
- `j` - View JSON output
- `a` - View aggregated data
- `q` - Quit

### 2. Get Current Readings (Python)

```python
from main import SensorSystem

system = SensorSystem()
system.start()

time.sleep(2)
data = system.get_current_data()
print(data)
# Output: {'timestamp': '...', 'temperature': 22.5, 'humidity': 45.3, ...}

system.stop()
```

### 3. Get Aggregated Data (60-second window)

```python
aggregated = system.get_aggregated_data()
print(aggregated)
# Output: {
#   'temperature_avg': 22.3,
#   'humidity_avg': 45.2,
#   'pressure_avg': 1013.2,
#   'voc_avg': 120.5,
#   'voc_max': 150.0,
#   'voc_min': 100.0,
#   'voc_rate_change_ppb_per_minute': 2.5,
#   ...
# }
```

### 4. Check VOC Calibration Status

```python
dds2 = system.dds2_reader

print(f"Calibrated: {dds2.is_calibrated()}")
print(f"Warming up: {dds2.is_warming_up}")

if dds2.is_warming_up:
    remaining = dds2.get_warmup_time_remaining()
    print(f"Warm-up remaining: {remaining} seconds")
```

## API Integration

### Connect to Flask Backend

```python
from flask import Flask, jsonify
from main import SensorSystem

app = Flask(__name__)
system = SensorSystem()

@app.route('/api/sensor/current', methods=['GET'])
def get_current_readings():
    data = system.get_current_data()
    return jsonify(data)

@app.route('/api/sensor/aggregated', methods=['GET'])
def get_aggregated():
    data = system.get_aggregated_data()
    return jsonify(data)

@app.route('/api/sensor/status', methods=['GET'])
def get_status():
    return jsonify(system.get_status())

if __name__ == '__main__':
    system.start()
    app.run(host='0.0.0.0', port=3001)
    system.stop()
```
